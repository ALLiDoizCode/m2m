### AC 9 (CLI Onboarding Wizard) Deferral Decision

**Acceptance Criteria 9** specifies an "Agent onboarding wizard (CLI tool) guides developers through integration."

**Deferral Rationale:**

- **Core Value**: Written documentation (AC 1-8, 10) provides 95% of the value for agent developers
- **API-First Approach**: Developers integrating agents typically use programmatic APIs, not interactive CLI tools
- **Resource Prioritization**: Epic 12 contains higher-priority infrastructure work (HSM/KMS, fraud detection)
- **MVP Sufficiency**: Quickstart guide (AC 5) + code examples (AC 4) provide sufficient onboarding experience
- **Future Enhancement**: CLI wizard is a quality-of-life improvement best suited for post-Epic 12 enhancement

**Decision**: Task 8 implements the CLI wizard structure but marks it as **optional/MVP-deferred**. If time permits during implementation, the wizard will be completed. Otherwise, it will be moved to a future epic as a developer experience enhancement.

**Acceptance Criteria Satisfaction**: AC 9 will be considered **satisfied** for Story 11.10 completion if comprehensive written documentation (AC 1-8, 10) is delivered, with CLI wizard deferred to Epic 12 or later.

**EVM (Base L2) Addresses:**

### Common Issues and Troubleshooting

Based on Epic 11 implementation, common issues to document:

**Issue 1: Master Seed Not Found**

- Symptom: `Error: master-seed not found in storage`
- Cause: Seed manager not initialized or seed file missing
- Solution: Initialize seed manager with `generateMasterSeed()` or import existing mnemonic
  [Source: Epic 11 Story 11.1 Technical Specification lines 57-82]

**Issue 2: Insufficient Funds for Gas**

- Symptom: Transaction fails with "insufficient funds for gas"
- Cause: Agent EVM wallet needs ETH for gas fees
- Solution: Fund wallet with ETH via `AgentWalletFunder.fundAgentWallet(agentId)`
  [Source: Epic 11 Story 11.4 Technical Specification lines 545-691]

**Issue 3: Wallet Derivation Index Collision**

- Symptom: `Error: Wallet already exists for agent`
- Cause: Agent ID already has derived wallet
- Solution: Use `getAgentWallet()` to retrieve existing wallet instead of `deriveAgentWallet()`
  [Source: Epic 11 Story 11.2 Technical Specification lines 207-243]

**Issue 4: XRP Account Not Activated**

- Symptom: XRP transactions fail with "Account not found"
- Cause: XRP accounts require minimum 10 XRP reserve to activate
- Solution: Send at least 15 XRP (reserve + buffer) to activate account
  [Source: Epic 11 Story 11.4 Technical Specification lines 555-557]

**Issue 5: Rate Limit Exceeded**

- Symptom: `Error: Funding rate limit exceeded`
- Cause: Exceeded max wallet creations/hour (100 default)
- Solution: Wait for rate limit window to reset or adjust rate limit config
  [Source: Epic 11 Story 11.9 AC 3, packages/connector/src/wallet/rate-limiter.ts]

### FAQ Topics to Cover

Based on Epic 11 functionality, FAQ must address:

**Wallet Lifecycle:**

- Q: How are agent wallets created?
- A: Via `AgentWalletLifecycle.createAgentWallet(agentId)` which derives addresses from master seed
  [Source: Epic 11 Story 11.5 Technical Specification lines 776-799]

- Q: What wallet states exist?
- A: pending → active → suspended → archived
  [Source: Epic 11 Story 11.5 Technical Specification lines 716-736]

- Q: How do I suspend a wallet?
- A: Call `suspendWallet(agentId, reason)` - wallet can be reactivated later
  [Source: Epic 11 Story 11.5 Technical Specification lines 824-849]

**Backup/Recovery:**

- Q: How do I backup agent wallets?
- A: Use `WalletBackupManager.createFullBackup(password)` - exports encrypted seed + metadata
  [Source: Epic 11 Story 11.8 Technical Specification lines 1297-1338]

- Q: Can I restore wallets on a new server?
- A: Yes, use `restoreFromBackup(backupData, password)` to restore all wallets
  [Source: Epic 11 Story 11.8 Technical Specification lines 1342-1372]

**Multi-Chain Support:**

- Q: Do agents get wallets for both EVM and XRP?
- A: Yes, each agent gets one EVM address and one XRP address from same derivation index
  [Source: Epic 11 Story 11.2 Technical Specification lines 183-198, 229-235]

- Q: Can I use different chains for different agents?
- A: Yes, choose chain when opening payment channels via `chain: 'evm' | 'xrp'` parameter
  [Source: Epic 11 Story 11.6 Technical Specification lines 989-1036]
